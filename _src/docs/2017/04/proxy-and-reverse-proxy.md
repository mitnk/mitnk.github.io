什么是反向代理
==============

大家应该都知道  
Nginx的通常角色是被用作反向代理  
你可能觉得你大概知道它是什么意思  
但你有对它的精切定义吗？  
但什么是反向代理？  
与普通代理或正向代理有何不同呢？  

（正向）代理
------------

正向代理是代理以你（client）的身份访问目标机器  
Proxy代表的是你（client）  
对于目标机器来说，他并不知道你的存在  
而对你（client）来说你即知道Proxy也知道目标机器  
一个例子是你用一个国外HTTP代理访问Facebook网站  

反向代理
--------

反向代理从字面就可以大概知晓  
是反着做代理的  
对于你（Client）来说  
你并不知道（或无从知道）目标机器的存在  
你访问的资源都像是Proxy本身直接提供的  
Proxy代表的是目标机器（群）  
像Nginx、HAProxy都是反向代理的例子
